# Описание программы (main.py)

## 1. Назначение
Программа представляет собой графическое приложение (Tkinter) для поиска **локального минимума** выбранной целевой функции с помощью:
- **генетического алгоритма (GA)**;
- **роевого алгоритма частиц (PSO)**.

Приложение поддерживает две целевые функции:
1) **1D**: \( f(x) = x\sin(x) \) на отрезке \([0, 20]\).
2) **2D**: \( f(x, y) = -0.0001\cdot |\sin(x)\sin(y)\exp(|100 - \sqrt{x^2+y^2}/\pi|) + 1|^{0.1} \) на квадрате \([-10, 10]\times[-10, 10]\).

## 2. Используемые библиотеки
- **tkinter**, **tkinter.ttk** — создание графического интерфейса.
- **numpy** — векторные вычисления, генерация случайных значений, вычисление целевых функций.
- **matplotlib** (backend **TkAgg**) — визуализация: графики функции, точки популяции/роя, график сходимости.

## 3. Целевые функции и границы
### 3.1. 1D функция
- Константы: `X_MIN = 0.0`, `X_MAX = 20.0`.
- Функция: `f(x)` — возвращает `x * sin(x)`.

### 3.2. 2D функция
- Константы: `X2_MIN = -10.0`, `X2_MAX = 10.0`, `Y2_MIN = -10.0`, `Y2_MAX = 10.0`.
- Функция: `f2(x, y)` — вычисляет значение 2D функции (векторизовано через NumPy).

## 4. Генетический алгоритм (GA)
Реализованы две версии: для 1D (оптимизация `f(x)`) и для 2D (оптимизация `f2(x,y)`). Во всех случаях выполняется **минимизация**.

### 4.1. Вспомогательная функция выбора снимков
#### Функция `_snapshot_indices(total, max_snapshots=8)`
- **Тип**: приватная (служебная)
- **Назначение**: вычисляет индексы поколений/итераций, на которых сохраняются «снимки» популяции/роя для визуализации.

### 4.2. GA для 1D
#### Функция `genetic_algorithm(pop_size, generations, crossover_prob, mutation_prob, mutation_scale)`
- **Тип**: публичная
- **Назначение**: минимизирует `f(x)` на `[X_MIN, X_MAX]`.
- **Представление особи**: одно вещественное число `x`.
- **Инициализация**: случайная популяция `population ~ U(X_MIN, X_MAX)`.
- **Оценка приспособленности**: `fitness = f(population)` (меньше — лучше).
- **Элитизм**: лучший `x` копируется в новую популяцию.
- **Отбор**: турнирный (размер турнира = 3), выбираются 2 родителя.
- **Кроссовер**: **арифметический** (выпуклая комбинация):
  - `child = alpha * parent_a + (1 - alpha) * parent_b`, где `alpha ~ U(0,1)`.
- **Мутация**: добавление гауссовского шума `N(0, mutation_scale)` с вероятностью `mutation_prob`.
- **Ограничение области поиска**: `np.clip(child, X_MIN, X_MAX)`.
- **Возвращаемое значение**:
  - `best_x` — найденная координата;
  - `best_y` — значение функции `f(best_x)`;
  - `best_history` — история лучших значений по поколениям;
  - `snapshots` — снимки популяции для графика.

### 4.3. GA для 2D
#### Функция `genetic_algorithm_2d(pop_size, generations, crossover_prob, mutation_prob, mutation_scale)`
- **Тип**: публичная
- **Назначение**: минимизирует `f2(x, y)` на `[X2_MIN, X2_MAX] × [Y2_MIN, Y2_MAX]`.
- **Представление особи**: пара вещественных координат `(x, y)`.
- **Инициализация**: `pop_x`, `pop_y` — независимые равномерные выборки по заданным границам.
- **Оценка**: `fitness = f2(pop_x, pop_y)`.
- **Элитизм**: лучший `(bx, by)` переносится в новое поколение.
- **Отбор**: турнирный (размер 3) отдельно по индексам.
- **Кроссовер**: арифметический для каждой координаты.
- **Мутация**: гауссовский шум для `x` и `y`.
- **Ограничение**: `np.clip` по каждой координате.
- **Возвращаемое значение**:
  - `best_x`, `best_y`;
  - `best_val = f2(best_x, best_y)`;
  - `best_history`;
  - `snapshots`.

## 5. Роевой алгоритм частиц (PSO)
Реализованы две версии: 1D и 2D. Во всех случаях выполняется **минимизация**.

Ключевая модификация: **ограничение скорости (velocity clamping)**, при этом `vmax` **линейно уменьшается** по итерациям, но не опускается ниже 5% от начального значения.

### 5.1. PSO для 1D
#### Функция `pso(swarm_size, iterations, c1, c2, w, vmax)`
- **Тип**: публичная
- **Назначение**: минимизирует `f(x)` на `[X_MIN, X_MAX]`.
- **Состояние частицы**:
  - `pos` — позиция (координата x);
  - `vel` — скорость;
  - `pbest_pos`, `pbest_val` — личный лучший результат.
- **Глобальный лучший**: `gbest_pos` по минимальному `pbest_val`.
- **Обновление скорости**:
  - `vel = w*vel + c1*r1*(pbest_pos - pos) + c2*r2*(gbest_pos - pos)`.
- **Ограничение скорости**:
  - `current_vmax = max(vmax*(1 - i/iterations), 0.05*vmax)`;
  - `vel = clip(vel, -current_vmax, current_vmax)`.
- **Обновление позиции**: `pos = clip(pos + vel, X_MIN, X_MAX)`.
- **Возвращаемое значение**:
  - `gbest_pos`, `gbest_val`;
  - `best_history`;
  - `snapshots`.

### 5.2. PSO для 2D
#### Функция `pso_2d(swarm_size, iterations, c1, c2, w, vmax)`
- **Тип**: публичная
- **Назначение**: минимизирует `f2(x, y)` на `[X2_MIN, X2_MAX] × [Y2_MIN, Y2_MAX]`.
- **Состояние роя**:
  - позиции: `pos_x`, `pos_y`;
  - скорости: `vel_x`, `vel_y`;
  - личные лучшие: `pbest_x`, `pbest_y`, `pbest_val`;
  - глобальный лучший: `gbest_x`, `gbest_y`.
- **Обновление скорости** выполняется отдельно для `x` и `y` по формуле PSO.
- **Ограничение скорости**: аналогично 1D через `current_vmax`.
- **Ограничение координат**: `clip` по каждой координате.
- **Возвращаемое значение**:
  - `gbest_x`, `gbest_y`, `gbest_val`;
  - `best_history`;
  - `snapshots`.

## 6. Графический интерфейс (GUI)
GUI состоит из главного окна и окна параметров.

### 6.1. Класс `App`
#### Методы
- `__init__()` — создает главное окно приложения.
- `_build_main()` — строит интерфейс выбора:
  - целевой функции (1D или 2D);
  - метода оптимизации (GA или PSO).
- `_open_ga()` — открывает окно параметров для GA.
- `_open_pso()` — открывает окно параметров для PSO.

### 6.2. Класс `ParamWindow`
Окно ввода параметров, запуска вычислений и отображения результатов.

#### Атрибуты (константы параметров)
- `GA_PARAMS` — список параметров GA (название, ключ, значение по умолчанию).
- `PSO_PARAMS` — список параметров PSO (название, ключ, значение по умолчанию).

#### Методы
- `__init__(parent, method, func="1d")` — инициализация окна, сохранение выбранных режима и функции.
- `_build()` — формирует элементы ввода параметров и область для графиков.
- `_get(key, cast=float)` — чтение значения параметра из поля ввода и приведение типа.
- `_run()` — запуск выбранного алгоритма:
  - для 1D вызывает `genetic_algorithm()` или `pso()`;
  - для 2D вызывает `genetic_algorithm_2d()` или `pso_2d()`;
  - выводит найденный минимум и строит графики.
- `_draw(best_x, best_y, history, snapshots)` — визуализация для 1D:
  - график `f(x)`;
  - точки популяции/роя на нескольких поколениях/итерациях (градиент цвета);
  - график сходимости.
- `_draw_2d(best_x, best_y, best_val, history, snapshots)` — визуализация для 2D:
  - контурная карта `f2(x,y)`;
  - точки популяции/роя по поколениям/итерациям;
  - график сходимости.

## 7. Точка входа
В конце файла расположен блок:
- `if __name__ == "__main__":` — создает объект `App()` и запускает `mainloop()`.
