## 1. Цель работы

**Основная цель:** разработать программу с графическим интерфейсом для поиска **локального минимума** заданной целевой функции методами **генетического алгоритма (ГА)** и **Particle Swarm Optimization (PSO)**.

**Конкретные задачи:**

1. Реализовать две целевые функции:
   - 1D: \( f(x) = x \cdot \sin(x) \) на отрезке \([0, 20]\);
   - 2D:  
     \[
     f(x, y) = -0.0001 \cdot \left|\sin(x)\sin(y) \cdot \exp\left(\left|100 - \frac{\sqrt{x^2+y^2}}{\pi}\right|\right) + 1\right|^{0.1}
     \]
     на области \([-10, 10] \times [-10, 10]\).
2. Реализовать ГА для 1D и 2D задач:
   - отбор турниром;
   - **арифметический кроссовер** как модификация;
   - мутация добавлением гауссовского шума;
   - элитизм.
3. Реализовать PSO для 1D и 2D задач:
   - учет личного и глобального лучшего;
   - **ограничение скорости (velocity clamping)** с линейным уменьшением \(V_{max}\) как модификация.
4. Разработать GUI на `tkinter`:
   - выбор целевой функции (1D/2D);
   - выбор метода оптимизации (ГА/PSO);
   - ввод параметров алгоритма;
   - визуализация результата (положение минимума и график сходимости).
5. Провести исследование влияния параметров алгоритмов на скорость сходимости и стабильность результата.

## 2. Теоретическая часть

### 2.1. Постановка задачи оптимизации

Задача заключается в поиске значения (или пары значений) аргумента, при которых целевая функция принимает минимальное значение:

- **1D:** найти \(x^*\), минимизирующее \(f(x)\) на \([X_{min}, X_{max}]\);
- **2D:** найти \((x^*, y^*)\), минимизирующее \(f(x, y)\) на прямоугольной области \([X2_{min}, X2_{max}] \times [Y2_{min}, Y2_{max}]\).

Поскольку функции нелинейны и имеют множество локальных экстремумов, применяются эвристические методы глобальной оптимизации.

### 2.2. Генетический алгоритм (ГА)

ГА имитирует процессы естественного отбора:

1. **Инициализация** популяции случайными решениями.
2. **Оценка приспособленности** (fitness) — вычисление значения целевой функции.
3. **Отбор** — в программе используется **турнирный отбор** (размер турнира 3).
4. **Скрещивание (кроссовер)** — реализован **арифметический кроссовер**:
   \[
   child = \alpha \cdot parent_a + (1-\alpha) \cdot parent_b, \quad \alpha \in [0,1]
   \]
5. **Мутация** — добавление нормального шума \(\mathcal{N}(0, \sigma)\).
6. **Элитизм** — лучший индивид переносится в новое поколение без изменений.

### 2.3. Роевой алгоритм (PSO)

PSO моделирует движение роя частиц в пространстве решений:

- каждая частица имеет **позицию** и **скорость**;
- учитывается личное лучшее положение \(pbest\) и глобальное лучшее \(gbest\);
- обновление скорости (1D векторно, 2D покомпонентно):

\[
\vec{v} \leftarrow w\vec{v} + c_1 r_1 (\vec{pbest} - \vec{x}) + c_2 r_2 (\vec{gbest} - \vec{x})
\]
\[
\vec{x} \leftarrow \vec{x} + \vec{v}
\]

**Модификация в работе:** ограничение скорости \(V_{max}\) с линейным уменьшением по итерациям (чтобы ускорять поиск на старте и стабилизировать к концу).

### 2.4. Визуализация и оценка сходимости

Для оценки процесса оптимизации строится **история лучшего значения** на каждой итерации/поколении. Это позволяет:

- сравнить скорость сходимости;
- выявить преждевременную сходимость (для ГА);
- увидеть стабилизацию результата (для PSO).

## 3. Основные шаги программы

### Шаг 1. Выбор задачи
Пользователь выбирает:
- целевую функцию (1D или 2D);
- метод оптимизации (ГА или PSO).

### Шаг 2. Ввод параметров
В отдельном окне вводятся параметры:

**ГА:**
- размер популяции (`pop_size`);
- число поколений (`generations`);
- вероятность кроссовера (`crossover_prob`);
- вероятность мутации (`mutation_prob`);
- масштаб мутации (`mutation_scale`).

**PSO:**
- размер роя (`swarm_size`);
- число итераций (`iterations`);
- коэффициенты \(c1, c2\);
- инерция \(w\);
- максимальная скорость (`vmax`).

### Шаг 3. Запуск алгоритма
- для 1D используются `genetic_algorithm()` или `pso()`;
- для 2D используются `genetic_algorithm_2d()` или `pso_2d()`.

### Шаг 4. Сохранение «снимков» популяции/роя
Для визуализации выбираются несколько поколений/итераций (до 8), чтобы:
- показать распределение особей/частиц по этапам;
- отобразить динамику приближения к минимуму.

### Шаг 5. Отображение результата
- текстовый вывод найденного минимума;
- график функции и точек популяции/роя;
- график сходимости.

## 5. Описание программы

Программа реализована в одном файле `main.py` и включает:

1. **Описание целевых функций** `f(x)` и `f2(x, y)` и диапазонов поиска.
2. **Реализацию алгоритмов:**
   - ГА (1D и 2D);
   - PSO (1D и 2D).
3. **GUI на `tkinter`:**
   - главное окно выбора;
   - окно параметров;
   - область построения графиков (через `matplotlib` и `FigureCanvasTkAgg`).

### 5.1. Целевые функции

- 1D функция: \(f(x) = x\sin(x)\) на \([0, 20]\).
- 2D функция (с экспонентой и синусами): используется как пример сложной многомодальной поверхности.

### 5.2. Генетический алгоритм

Особенности реализации:
- отбор: турнирный (3 участника);
- кроссовер: арифметический (выпуклая комбинация родителей);
- мутация: гауссовский шум с параметром `mutation_scale`;
- элитизм: перенос лучшего решения.

### 5.3. PSO

Особенности реализации:
- линейное уменьшение `vmax` по итерациям до 5% от исходного значения;
- обновление личных и глобальных лучших значений.

### 5.4. Визуализация

**Для 1D:**
- график функции;
- точки популяции/роя на выбранных итерациях (градиент цветов);
- точка найденного минимума (звезда);
- график сходимости.

**Для 2D:**
- контурный график `contourf`;
- точки популяции/роя;
- точка минимума;
- график сходимости.

## 10. Вывод

В ходе работы реализована программа с графическим интерфейсом для решения задачи оптимизации (поиска локального минимума) в 1D и 2D постановках.

Программа позволяет интерактивно менять параметры, наблюдать визуализацию популяции/роя и анализировать процесс сходимости по графикам, что делает её удобным инструментом для изучения эвристических методов оптимизации.
